Bubble Up avec Try/Catch

Comprendre le principe du Bubble Up avec try...catch en JavaScript

Dans le développement d'applications JavaScript, la gestion des erreurs est un aspect essentiel pour garantir une expérience utilisateur fluide et une application stable. Ce cours se concentre sur un comportement fondamental lié au bloc try...catch : le bubble up (ou remontée des erreurs).

Dernière mise à jour : 20.09.2025

Introduction

Comprendre cette notion permet de mieux structurer ses applications, éviter les erreurs silencieuses et centraliser la gestion des exceptions.

Objectifs pédagogiques

  • Comprendre le comportement d’une erreur non capturée immédiatement.
  • Décrire le mécanisme de "remontée" (bubble up) d’une erreur dans la pile d’appels.
  • Implémenter une stratégie efficace de gestion d’erreurs dans une application JavaScript.

Définitions clés

  • Erreur (Error) : Objet généré lorsqu’une situation inattendue ou bloquante survient pendant l’exécution du code.
  • try...catch : Bloc permettant d’essayer un morceau de code (try) et de capturer toute erreur générée (catch).
  • Bubble up : Mécanisme naturel par lequel une erreur "remonte" la pile d’exécution jusqu’à ce qu’elle soit interceptée par un bloc catch.

Exemple illustratif

function lireFichier() {
  throw new Error("Impossible de lire le fichier !");
}

function traiterFichier() {
  lireFichier(); // pas de try...catch ici
}

function lancerTraitement() {
  try {
    traiterFichier();
  } catch (e) {
    console.log("Erreur interceptée :", e.message);
  }
}

lancerTraitement();

Ce qui se passe :

  • lancerTraitement() appelle traiterFichier().
  • traiterFichier() appelle lireFichier().
  • lireFichier() provoque une erreur avec throw.
  • Aucune des fonctions intermédiaires n’intercepte cette erreur.
  • L’erreur "bulle" jusqu’à la fonction lancerTraitement(), qui la capture.

Bonnes pratiques

  • Éviter de multiplier les blocs try...catch à tous les niveaux : cela rend le code verbeux et difficile à maintenir.
  • Préférer capturer les erreurs au plus haut niveau logique (ex: dans un contrôleur ou une fonction principale).
  • Toujours afficher ou enregistrer l’erreur pour faciliter le débogage (console.error, logs, etc).
  • Utiliser finally pour exécuter du code quel que soit le résultat (try ou catch).

Variante avec async/await

async function chargerRessource() {
  throw new Error("Erreur de chargement");
}

async function main() {
  try {
    await chargerRessource();
  } catch (e) {
    console.error("Erreur asynchrone :", e.message);
  }
}

main();

Même en asynchrone, le principe de bubble up reste le même : si tu ne catch pas une erreur awaitée, elle remonte jusqu'à un bloc supérieur.

Résumé

Élément Rôle
throw Déclenche une erreur
try Contient le code "à risque"
catch Capture et gère l’erreur
Bubble up Laisse l’erreur remonter jusqu’à un bloc catch approprié
